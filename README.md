# Question Dimension-Up Skill

一个用于审计、评分、回答并升维商业问题的 Codex Skill。

它的核心目标不是让 AI 更快回答问题，而是先判断：**这个问题本身是否值得被回答**。如果问题停留在信息检索、动作技巧、模糊趋势或错误前提上，Skill 会先指出问题质量，再把它重构为更接近商业本质的研究问题。

## 适用场景

- 行业研究问题诊断
- 创业、投资、战略、商业模式分析
- 创始人内容和个人品牌问题拆解
- 专家访谈问题设计
- AI 产品、SaaS、消费品牌、平台型业务的商业判断
- 判断一个问题是真问题、伪问题、好问题还是差问题

## 输出结构

每次回答固定使用六段结构：

1. 问题得分
2. 分析问题
3. 引申问题
4. 回答
5. 问题升维
6. 追问

## 核心方法论

这个 Skill 借鉴并组合了以下战略分析框架：

- 麦肯锡假设驱动法：判断问题是否包含可验证的商业假设
- MECE 原则：判断问题是否有结构化拆解能力
- 3C 战略框架：从客户、公司、竞争对手分析问题
- 价值链拆解：识别行业上下游与关键控制点
- 利润池分析：判断钱、利润和现金流沉淀在哪里
- 定价权分析：判断谁真正掌握商业权力
- KSF 与极端场景推演：判断压力测试下谁能活下来

## 严格评分机制

Skill 会按 100 分制评分：

- 假设性 Hypothesis：30 分
- 结构度 Structure：30 分
- 洞察力 Insight：40 分

评级标准：

- 0-30：青铜级，初级/资料搜集型
- 31-50：白银级，表层业务问题
- 51-70：黄金级，业务关系型
- 71-90：钻石级，商业洞察型
- 91-100：王者级，终局推演型

评分口径故意偏严格：

- 没有具体商业变量的问题，原则上不超过 50 分。
- 没有假设和结构的问题，原则上不超过 70 分。
- 没有触及利润池、定价权、KSF、终局或极端场景的问题，原则上不超过 85 分。

## 问题性质判断

当问题质量较低时，Skill 会直接判断问题弱在哪里：

- 行业理解不足：缺少行业结构、价值链、关键玩家、商业模式背景
- 提问思维不足：问题模糊、表层、无假设、无结构、不可验证
- 理解错误：问题建立在错误前提、错误因果、错误指标或概念混淆上

低分问题不会被鼓励式包装。Skill 会直接指出：

- 这不是一个好问题。
- 这是一个比较初级的问题。
- 这是一个偏小白的问题。
- 这个问题建立在一个可能错误的理解上。

## 安装

将 `skills/question-dimension-up` 复制到你的 Codex skills 目录：

```bash
cp -R skills/question-dimension-up ~/.codex/skills/
```

然后在新的 Codex 会话中使用类似问题触发：

```text
我要启动提问升维。我的原始问题是：很多 AI 初创公司为什么会在平台期死掉？
```

或直接提问：

```text
创始人如何在 Twitter 上做爆款内容？
```

## 设计原则

这个 Skill 的核心判断是：

> 好问题不是为了得到答案。好问题本身，就是理解行业的入口。

它会推动问题从：

- 现象到机制
- 动作到变量
- 流量到商业资产
- 增长到利润
- 产品功能到工作流控制点
- 公司表现到行业终局
